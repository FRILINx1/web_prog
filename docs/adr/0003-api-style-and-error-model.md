# 0003-api-style-and-error-model: Стиль API та формат помилок

| Атрибут | Значення |
| :--- | :--- |
| **Дата** | 2025-12-03 |
| **Статус** | Прийнято |
| **Стиль API** | **RESTful API**. Використання стандартних HTTP-методів (GET, POST, PATCH, DELETE) та ресурсних іменників (`/tasks`). |
| **Контракт** | Публічний API суворо відповідає специфікації у `docs/api/openapi.yaml`. |
| **Коди відповіді** | Використання стандартних кодів: 200 (ОК), 201 (Створено), 204 (DELETE), 400 (Валідація), 401 (Неавторизований), 404 (Не знайдено). |
| **Формат помилок** | Усі помилки повинні відповідати єдиній схемі **ErrorResponse** (див. `openapi.yaml`) для забезпечення узгодженості. |

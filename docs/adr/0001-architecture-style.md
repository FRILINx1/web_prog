# 0001-architecture-style: Вибір модульного моноліту

| Атрибут | Значення |
| :--- | :--- |
| **Дата** | 2025-12-03 |
| **Статус** | Прийнято |
| **Контекст** | Проєкт є невеликим веб-додатком **MyTaskApp** з двома основними контекстами: Автентифікація та Керування Завданнями. Команда невелика, розгортання та інфраструктура повинні бути максимально простими. |
| **Рішення** | Використовувати архітектурний стиль **Модульний Моноліт**. Код буде логічно розділений на внутрішні модулі (API, Service, Domain), але розгортатиметься як єдиний застосунок. |
| **Альтернативи** | Мікросервіси. |
| **Наслідки** | Спрощене розгортання, єдина база даних (легкість транзакцій), простіший моніторинг, швидший розвиток на початкових етапах. |
| **Умови перегляду** | Рішення буде переглянуто, якщо: 1) Команда зросте до 6+ розробників. 2) Виникне необхідність масштабувати контекст Task Management незалежно від Authentication. |
